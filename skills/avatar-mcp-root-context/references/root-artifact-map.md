# Root Artifact Map

Read this file when the task needs the rationale behind the zero-shot packet, the payload shapes for protected MCP tools, or the shortest path to the root repo artifacts.

## Root Guidance

- `AGENTS.md`
  Primary repo workflow, testing, and frontier-agent guidance.
- `AVATAR_SYSTEM.md`
  Avatar and Judge integration summary.
- `avatar_bindings.v1.json`
  Avatar identities, vessels, capsules, and visual bindings.

## Frontier And RBAC Artifacts

- `registry/agents/frontier_agent_index.v1.json`
  Current frontier-agent inventory plus MCP tool list.
- `registry/agents/frontier_agent_cards.v1.json`
  Current role, model, embedded-skill, and MCP-tool scope per frontier agent.
- `runtime/rbac/frontier_rbac_tokens.local.json`
  Local token bundle generated by `make frontier-index`; treat as secret and do not commit it.

Regenerate the frontier artifacts with:

```bash
make frontier-index
```

Fallback:

```bash
python scripts/build_frontier_agent_index.py
```

## MCP Launch Surfaces

- `app/mcp_tooling.py`
  Protected tool ingress for `ingest_repository_data` and `ingest_avatar_token_stream`, plus `build_local_world_foundation_model` and `get_coding_agent_avatar_cast`.
- `runtime_mcp_server.py`
  Runtime MCP node surface for `route_a2a_intent`, embed tools, and runtime assignment tools.
- `embed_control_plane.py`
  Deterministic A2A-to-MCP control-plane logic and receipt chaining.
- `mcp_server.py`
  Stdio compatibility entrypoint; use only when the client explicitly needs stdio transport.

## Minimal Payload Rules

- `build_local_world_foundation_model`
  Include `prompt`, `repository`, `commit_sha`, `actor`, `cluster_count`, and `risk_profile`.
- `ingest_repository_data`
  Include a root-repo snapshot and `authorization: "Bearer <OIDC token>"`.
  The snapshot repository must match the verified repository claim.
- `ingest_avatar_token_stream`
  Include `payload.namespace`, `payload.max_tokens`, and `payload.tokens`, plus `authorization: "Bearer <OIDC token>"`.
- `route_a2a_intent`
  Call after the repository snapshot and avatar token stream are prepared.

## Zero-Shot Packet Rules

- Keep the packet concise and rooted in the repo root.
- Prefer artifact references, counts, IDs, and hashes over long prose dumps.
- Keep secrets and live bearer tokens out of stored artifacts.
- If the downstream model needs deeper code context, add file references from the packet instead of embedding whole source files.
